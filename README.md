# ⚽ Турнир во мал фудбал „БОРИС ТРАЈКОВСКИ“ — Моноспитово

Веб страница за локален турнир со **16 тима** во **4 групи** + **елиминации**.
Изработена со **React (Vite)**, **Supabase** и **Tailwind CSS**. Целосно на македонски.

## Што овозможува

**Јавен дел (за секого):**
- 📋 **Резултати** — сите мечеви (во тек / претстојни / одиграни) со стрелци под секој меч
- 🏆 **Групи** — табели по 4-те групи (бодови, гол-разлика) — се пресметуваат автоматски
- 🎯 **Елиминации** — четвртфинале · полуфинале · финале
- 👥 **Тимови** — сите тимови и играчи (до 10 по тим)
- 👟 **Стрелци** — листа на најдобри стрелци (се ажурира сама од внесените голови)

**Админ дел (само со најава):**
- Креирање и уредување на тимови и распоред по групи
- Пополнување играчи во секој тим (до 10)
- Креирање мечеви, внесување и менување резултати и статус
- Внесување голови (кој играч, која минута) под секој меч

## 🚀 Подесување чекор-по-чекор

### 1. Инсталирај ги пакетите
```bash
npm install
```

### 2. Направи Supabase проект
1. Оди на [supabase.com](https://supabase.com) и најави се (бесплатно).
2. **New project** → внеси име и лозинка за базата → **Create**.

### 3. Направи ги табелите
1. Во Supabase, лево избери **SQL Editor** → **New query**.
2. Отвори ја датотеката [`supabase/schema.sql`](supabase/schema.sql) од овој проект,
   копирај ја **целата** содржина, залепи ја и притисни **Run**.
   > Ова ги прави табелите, автоматските табели/стрелци и безбедносните правила.

### 4. Поврзи ја апликацијата со Supabase
1. Во Supabase: **Project Settings** (долу лево) → **API**.
2. Копирај ги **Project URL** и **anon public** клучот.
3. Во овој проект, преименувај го `.env.example` во `.env` (или отвори го постоечкиот `.env`)
   и внеси ги вредностите:
   ```
   VITE_SUPABASE_URL=https://xxxxx.supabase.co
   VITE_SUPABASE_ANON_KEY=eyJhbGciOi...
   ```

### 5. Направи админ корисник (организаторот)
1. Во Supabase: **Authentication** → **Users** → **Add user** → **Create new user**.
2. Внеси **емаил** и **лозинка** (ова се податоците за најава во админ панелот).
3. Чекирај „Auto Confirm User“ ако постои опцијата.

### 6. Стартувај ја страницата
```bash
npm run dev
```
Отвори го линкот што ќе се појави (обично `http://localhost:5173`).
Најави се во админ преку копчето **Админ** горе десно.

## 📖 Како се користи (за админот)

1. **Тимови** → додај ги 16-те тима и стави ги во групи A, B, C, D (по 4 во секоја).
2. **Играчи** → избери тим и додај играчи (до 10).
3. **Мечеви** → креирај мечеви (групна фаза или елиминации), внеси резултат и статус
   (*Закажан / Во тек / Завршен*).
4. **Голови** → избери меч и додади кој играч колку голови дал — листата на стрелци
   и резултатите на јавната страница се ажурираат автоматски.

> 💡 Табелите по групи се пресметуваат **само од завршени** групни мечеви.
> Стави го статусот на **Завршен** за мечот да се смета во табелата.

## 🌍 Објавување онлайн (кога ќе бидеш подготвен/а)

Најлесно преку **Vercel** или **Netlify** (бесплатно):
1. Качи го проектот на GitHub.
2. Поврзи го репото на [vercel.com](https://vercel.com).
3. Во поставките за проектот додај ги истите две променливи од `.env`
   (`VITE_SUPABASE_URL`, `VITE_SUPABASE_ANON_KEY`).
4. Deploy → добиваш јавен линк што можеш да го споделиш.

## 🛠️ Команди
| Команда | Значење |
|---|---|
| `npm run dev` | стартува локален сервер за развој |
| `npm run build` | прави продукциска верзија во `dist/` |
| `npm run preview` | прегледува ја продукциската верзија локално |

## Структура
```
supabase/schema.sql     ← база (табели, погледи, безбедност)
src/lib/                ← Supabase клиент, константи, прашалници
src/context/            ← најава (Auth)
src/components/          ← заеднички делови (навигација, картички, форми)
src/pages/               ← јавни страници
src/pages/admin/         ← админ панел (тимови, играчи, мечеви, голови)
```
